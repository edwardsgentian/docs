# Game Management

Organizers can create, edit, and manage games through the organizer dashboard.

## Creating Games

### From Organizer Dashboard

<Steps>
  <Step title="Navigate to Games Tab">
    In the organizer dashboard, click on the "Games" tab.
  </Step>
  <Step title="Click +Game">
    Click the "+Game" button in the header.
  </Step>
  <Step title="Game Creation Wizard">
    You'll be redirected to the game creation wizard.
  </Step>
</Steps>

### Game Creation Wizard

The wizard is a multi-step process that guides you through:

<CardGroup cols={2}>
  <Card title="Basic Info" icon="info">
    - **Name**: Game title (e.g., "Saturday Morning Game")
    - **Description**: Optional details about the game
    - **Location**: Full address with Google Maps integration
  </Card>
  <Card title="Schedule" icon="calendar">
    - **Date**: Game date (date picker)
    - **Time**: Start time (time picker)
    - **Duration**: How long the game lasts (in hours)
  </Card>
  <Card title="Pricing" icon="dollar-sign">
    - **Price**: Cost per spot (in dollars)
    - Supports decimal values (e.g., $15.50)
  </Card>
  <Card title="Capacity" icon="users">
    - **Total Tickets**: Initial number of spots available
    - **Available Tickets**: Can be increased later for waitlist promotions
  </Card>
</CardGroup>

<Info>
After creating a game, you'll be redirected back to the organizer dashboard Games tab. The game will appear in the table immediately.
</Info>

## Editing Games

<Steps>
  <Step title="Find Game">
    Locate the game in the Games table.
  </Step>
  <Step title="Click Edit">
    Click the "Edit" button in the Actions column.
  </Step>
  <Step title="Update Fields">
    Modify any editable field in the modal.
  </Step>
  <Step title="Save Changes">
    Click "Save" to update the game.
  </Step>
</Steps>

## Editable Fields

All fields in the games table are editable through the Edit modal:

<CardGroup cols={2}>
  <Card title="Basic Information" icon="info">
    - **Name**: Game title
    - **Description**: Game description
    - **Location**: Full address (truncated in table if long)
  </Card>
  <Card title="Schedule" icon="calendar">
    - **Date**: Game date (date picker)
    - **Time**: Start time (time picker)
  </Card>
  <Card title="Pricing" icon="dollar-sign">
    - **Price**: Cost per spot (supports decimals)
  </Card>
  <Card title="Capacity" icon="users">
    - **Initial Allocation**: Original ticket count set when creating the game
    - **Current Allocation**: Current maximum capacity (can be increased for waitlist promotions)
  </Card>
</CardGroup>

### Capacity Constraints

<Warning>
You cannot reduce "Initial Allocation" or "Current Allocation" below the number of current attendees. The system validates this before saving.
</Warning>

<Info>
"Current Allocation" can be increased above "Initial Allocation" when promoting players from the waitlist. This allows organizers to accommodate more players than originally planned.
</Info>

## Deleting Games

<Steps>
  <Step title="Click Delete">
    Click the "Delete" button in the Actions column.
  </Step>
  <Step title="Confirm">
    Confirm deletion in the modal.
  </Step>
</Steps>

<Warning>
Deleting a game will remove all attendee records and cannot be undone.
</Warning>

## Capacity Management

### Initial vs Current Allocation

- **Initial Allocation**: The original number of tickets set when creating the game
- **Current Allocation**: Can be increased by organizers when promoting from waitlist

<Info>
When promoting waitlist players, the system automatically increases "Current Allocation" to accommodate them.
</Info>

## Waitlist Management

The waitlist system automatically manages players who want to join full games. As an organizer, you can view, manage, and promote waitlist players.

### Viewing the Waitlist

<Steps>
  <Step title="Open Game Details">
    Click on a game name in the Games table to view full game details.
  </Step>
  <Step title="Navigate to Waitlist">
    If the game is full, you'll see a "Waitlist" section showing all players waiting for a spot.
  </Step>
  <Step title="View Waitlist Details">
    The waitlist displays:
    - Player names and avatars
    - Position in queue (order they joined)
    - Date/time they joined the waitlist
    - Player level and positions (if available)
  </Step>
</Steps>

### Automatic Waitlist Promotion

The system automatically promotes waitlist players when spots become available:

<CardGroup cols={2}>
  <Card title="Attendee Cancellation" icon="user-x">
    When an attendee marks themselves as "not attending", the first waitlist player is automatically promoted
  </Card>
  <Card title="Capacity Increase" icon="users">
    When you increase "Current Allocation", waitlist players are automatically promoted to fill new spots
  </Card>
  <Card title="First Come, First Served" icon="clock">
    Promotions follow waitlist order - first player to join is first to be promoted
  </Card>
  <Card title="Email Notifications" icon="mail">
    Promoted players receive automatic email notifications with payment instructions
  </Card>
</CardGroup>

<Info>
Automatic promotion ensures fair, first-come-first-served access to available spots without requiring manual intervention.
</Info>

### Manual Waitlist Promotion

You can manually promote specific waitlist players:

<Steps>
  <Step title="View Waitlist">
    Navigate to the game details page and locate the Waitlist section.
  </Step>
  <Step title="Select Player">
    Find the player you want to promote in the waitlist table.
  </Step>
  <Step title="Click Promote">
    Click the "Promote" button next to the player's name.
  </Step>
  <Step title="Confirm Promotion">
    Confirm that you want to promote this player. The system will:
    - Move the player from waitlist to attendee list
    - Increase "Current Allocation" if needed
    - Send the player an email with payment instructions
  </Step>
</Steps>

<Warning>
Manually promoting a player may skip others who joined the waitlist earlier. Use this feature carefully to maintain fairness.
</Warning>

<Info>
When you manually promote a player, the system automatically increases "Current Allocation" to accommodate them if the game is at capacity.
</Info>

### Increasing Capacity for Waitlist

You can increase game capacity to accommodate waitlist players:

<Steps>
  <Step title="Edit Game">
    Click "Edit" on the game in the Games table.
  </Step>
  <Step title="Increase Current Allocation">
    Increase the "Current Allocation" field to a number higher than "Initial Allocation".
  </Step>
  <Step title="Save Changes">
    Click "Save" to update the game capacity.
  </Step>
  <Step title="Automatic Promotion">
    The system automatically promotes waitlist players (in order) to fill the new spots up to the new capacity.
  </Step>
</Steps>

<Info>
Increasing capacity is useful when you have many waitlist players and can accommodate more attendees than originally planned (e.g., larger field, additional time slot).
</Info>

### Waitlist Statistics

The waitlist provides insights into game demand:

- **Waitlist Size**: Number of players waiting for spots
- **Demand Indicator**: High waitlist numbers suggest you could create more games or increase capacity
- **Player Interest**: Shows which games are most popular

<Info>
Use waitlist data to inform future game planning - high waitlists suggest strong demand for similar games.
</Info>

### Managing Waitlist Players

You can view additional information about waitlist players:

- **Player Profile**: Click on a player's name to view their full profile
- **Game History**: See their participation history with your group
- **Contact Info**: Access contact information if needed (respect privacy)

<Warning>
Only promote waitlist players manually when you have a good reason (e.g., balancing teams, accommodating regular members). Default to automatic promotion for fairness.
</Warning>
